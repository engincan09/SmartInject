**SmartInject** is a zero-dependency, compile-time Dependency Injection source generator for .NET. 

It completely eliminates the need for reflection-based assembly scanning (like Scrutor) or maintaining a messy, thousand-line `Program.cs` file. By leveraging Roslyn Source Generators, SmartInject writes your DI container registrations at compile time, providing **maximum performance with zero startup cost**.

---

## ✨ Features

- ⚡ **Zero Startup Overhead:** No reflection, no runtime scanning. It generates standard `services.Add...` code behind the scenes.
- 📦 **Zero External Dependencies:** It doesn't pollute your project with external DLLs. The necessary attributes are injected directly into your source code during the build.
- 🧹 **Clean Architecture:** Say goodbye to endless `services.AddScoped<IInterface, Implementation>()` lines.
- 🎯 **Type-Safe:** Strongly typed attribute constructors (`typeof(IYourInterface)`) prevent runtime registration crashes.

## 📥 Installation

Install the package via NuGet Package Manager or .NET CLI:

```bash
dotnet add package SmartInject.Core

🚀 Quick Start
1. Decorate your service classes
Add the provided attributes to your implementation classes. You can choose between [Scoped], [Transient], or [Singleton].

using SmartInject;

// Standard web service
[Scoped(typeof(ITenantManager))]
public class TenantManager : ITenantManager 
{ 
    public void ConfigureTenant() { /* ... */ }
}

// Background listener or hardware integration
[Singleton(typeof(ISensorListener))]
public class TemperatureSensorListener : ISensorListener 
{ 
    public void StartListening() { /* ... */ }
}

// Lightweight stateless operations
[Transient(typeof(IEmailSender))]
public class EmailSender : IEmailSender 
{ 
    public void Send(string to) { /* ... */ }
}


2. Register them all with one line
Go to your Program.cs (or Startup.cs) and call the automatically generated extension method.

using SmartInject.Extensions; // Include the generated namespace

var builder = WebApplication.CreateBuilder(args);

// This single line registers all classes marked with SmartInject attributes across your project!
builder.Services.AddSmartInjectServices(); 

var app = builder.Build();
app.Run();


🛠️ How It Works (Under the Hood)
When you hit Build, SmartInject's incremental source generator kicks in:

It injects the [Scoped], [Transient], and [Singleton] attributes into your project transparently.

It scans your code for classes decorated with these attributes.

It generates an extension method named AddSmartInjectServices() that contains the native Microsoft.Extensions.DependencyInjection code.

If you look at your compiler output (under Dependencies -> Analyzers), you will see the exact generated C# code. It looks like this:

// <auto-generated />
using Microsoft.Extensions.DependencyInjection;

namespace SmartInject.Extensions
{
    public static class DependencyInjectionExtensions
    {
        public static IServiceCollection AddSmartInjectServices(this IServiceCollection services)
        {
            services.AddScoped<ITenantManager, TenantManager>();
            services.AddSingleton<ISensorListener, TemperatureSensorListener>();
            services.AddTransient<IEmailSender, EmailSender>();
            return services;
        }
    }
}

⚠️ Troubleshooting
Q: I added the attribute, but AddSmartInjectServices() is not showing up!
A: Visual Studio sometimes caches Source Generators. If IntelliSense is complaining:

Build the project once (Ctrl + Shift + B).

If it still persists, close and reopen your IDE, or restart the OmniSharp/Roslyn language server.

Q: Does it support Keyed Services (.NET 8)?
A: Not in the current release, but it is on the roadmap for future versions!
